# Ecosystem Parity

Once uses Buck2 and Bazel rules as behavioral references. It does not parse
their build files or reproduce transition, toolchain, and provider interfaces
verbatim. A parity field accepted by a Once target either affects declared
actions and providers or fails analysis with an explicit unsupported-field
diagnostic. Upstream fields that are not declared by a Once target remain
outside that target's supported boundary.

| Ecosystem | Once target kinds | Reference rules | Implemented parity | Intentional boundaries |
| --- | --- | --- | --- | --- |
| Rust | [`rust_library`](/reference/prelude/rust_library), [`rust_binary`](/reference/prelude/rust_binary), [`rust_test`](/reference/prelude/rust_test), [`rust_proc_macro`](/reference/prelude/rust_proc_macro) | [Buck2 Rust rules](https://buck2.build/docs/prelude/rules/rust/), [Bazel Rust rules](https://bazelbuild.github.io/rules_rust/rust.html) | Direct crate dependencies, dependency aliases, compiler environment files, compile and runtime data, feature flags, linker scripts, propagated native linker flags, procedural macros, executable runs, and test runs | Documentation and lint actions, native link dependency edges, alternate link styles, and Bazel's already-built crate test mode remain outside the supported boundary |
| Zig | [`zig_library`](/reference/prelude/zig_library), [`zig_c_library`](/reference/prelude/zig_c_library), [`zig_binary`](/reference/prelude/zig_binary), [`zig_static_library`](/reference/prelude/zig_static_library), [`zig_shared_library`](/reference/prelude/zig_shared_library), [`zig_test`](/reference/prelude/zig_test), and configured variants | [Bazel Zig rules](https://github.com/hermeticbuild/rules_zig/blob/main/docs/rules.md) | The complete core rule family, module imports and aliases, C translation, C sources and dependencies, runtime data, compiler and linker controls, auxiliary outputs, documentation, binary runs, test runs, and configured builds | Configured targets carry settings directly because Once does not need Bazel transition wrappers or an `actual` target indirection |
| Elixir | [`elixir_library`](/reference/prelude/elixir_library), [`elixir_test`](/reference/prelude/elixir_test) | [Bazel Elixir rules](https://github.com/rabbitmq/rules_elixir), [Buck2 Erlang application rules](https://buck2.build/docs/prelude/rules/erlang/erlang_app/) | Application metadata, dependencies, bytecode, protocol consolidation, configuration and private resources, compile and test options, explicit and inherited environment values, test setup, support tools, and ExUnit execution | Erlang source transforms, application templates, upgrade files, include-only dependencies, shell assembly, and `.ez` archive dependencies remain outside the supported boundary |
| Swift for Android | [`swift_android_library`](/reference/prelude/swift_android_library) | [Bazel Swift library](https://github.com/bazelbuild/rules_swift/blob/main/doc/rules.md#swift_library) | Module and shared-library compilation, package names, compiler inputs, conditional compilation symbols, transitive module inputs, propagated linker options, runtime data, and library-evolution interfaces | Static-link selection, compiler plugins, implementation-only dependencies, developer search paths, always-link behavior, and generated Objective-C headers remain outside the supported boundary |
| Android | [`android_resource`](/reference/prelude/android_resource), [`android_library`](/reference/prelude/android_library), [`android_binary`](/reference/prelude/android_binary), [`android_local_test`](/reference/prelude/android_local_test), [`android_instrumentation_test`](/reference/prelude/android_instrumentation_test) | [Buck2 Android rules](https://buck2.build/docs/prelude/rules/android/), [Bazel Android rules](https://bazel.build/reference/be/android) | Resource and asset compilation, Java and Kotlin compilation, Java compiler option aliases, library archives, runtime pruning, dexing, package assembly and signing, native libraries, local test controls, device test environment, support package installation, and test filters | Annotation processing, Android Interface Definition Language compilation, data binding, code shrinking, resource filtering, manifest placeholder expansion, and managed device provisioning remain outside the supported boundary |

The target kind schemas are the source of truth for the current boundary. Use
`once query schema <kind>` to inspect accepted fields, providers, capabilities,
and starter examples without reading prose documentation.
